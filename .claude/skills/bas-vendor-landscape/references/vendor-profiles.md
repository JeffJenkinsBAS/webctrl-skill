# BAS Vendor Profiles — Deep Detail

Per-vendor detail supporting the comparison table in the main SKILL.md. Use this when preparing a detailed competitive brief on one specific vendor, or when a technician needs deeper context on how a competing platform actually works.

---

## Siemens Desigo (Desigo CC + PXC controllers)
- **Front end**: Desigo CC — an open, cross-domain platform integrating BAS, fire, power, access, and video. Protocol support: BACnet, BACnet/SC, OPC DA/UA, Modbus, SNMP, KNX, IEC61850, HTML5, Haystack, S7/S7+; SDK available for subsystem extension. IT security aligned to ISA-99/IEC 62443 SL2. BTL-certified for B-XAWS/B-AWS/B-ALSWS/B-AACWS/B-RTR/B-BBMD profiles ([Siemens Desigo CC datasheet](https://cache.industry.siemens.com/dl/files/477/109791477/att_1048229/v7/A6V11852707.pdf)).
- **Controller programming**: PXC controllers use **D-MAP**, a proprietary graphical language with "close resemblance to CEN standard 11312," engineered through the proprietary **Desigo Xworks Plus** tool — not an EIKON-style universal microblock system ([PXC-D datasheet](https://www.intellisys.it/media/wysiwyg/Siemens_documentation/DESIGO/PXC_D_Eng_Datasheet.pdf)).
- **2024 updates**: BACnet Rev 1.16 and BACnet/SC support for Desigo Control Point/DRA, max 1,000 nodes/system, 100 nodes/hub for pure BACnet/SC deployments ([Siemens product update deck](https://assets.new.siemens.com/siemens/assets/api/uuid:de2d591f-ca68-426b-bc7c-5e7a610febb9/8-Termek-portfolio-valtozasai-Erhardt.pdf)).
- **Market position**: Global tier-1, strong in large/complex campus and mission-critical (data center, hospital) verticals; direct Siemens branches + dealer network similar to ALC.
- **Takeaway for ALC dealers**: Desigo CC's cross-domain (fire/security/video) breadth exceeds WebCTRL's BAS-only scope — don't compete there. Compete on controller-level troubleshooting speed (EIKON Logic tab vs. Xworks Plus separate tool session).

## Johnson Controls Metasys
- **Openness**: BACnet Protocol Revision 19 support; Metasys servers can act as BACnet Advanced Workstations, integrating third-party BACnet devices natively without extra gateway hardware ([JCI Metasys launch flyer](https://www.johnsoncontrols.com/-/media/project/jci-global/johnson-controls/us-region/united-states-johnson-controls/building-automation-and-controls/metasys/35359_jc_metasys-141-launch-flyer-buildingops_bms.pdf)).
- **Metasys 16.0 (2026)**: Adds a "Cyber Health Dashboard," IEEE 802.1x + BACnet/SC secure connectivity, **IEC 62443-4-2 SL2 compliance** (explicitly positioned toward 2027 EU Cybersecurity Resilience Act readiness), and AI-assisted low-code integration tooling ([JCI Metasys 16.0 announcement](https://www.johnsoncontrols.com/building-insights/thought-leadership/metasys-sets-a-new-standard-for-smarter-buildings)).
- BACnet/SC certificate lifecycle (CSR generation, CA signing, "Secure Connect Only Mode") is documented in detail in JCI's own configuration tool docs — notably more mature/public-facing than ALC's BACnet/SC documentation ([Metasys BACnet/SC management docs](https://docs.johnsoncontrols.com/bas/r/Metasys/en-US/Johnson-Controls-System-Configuration-Tool-JCT-Help/6.0/Infrastructure/Security-and-authorization/BACnet/SC-Management)).
- **Also runs Niagara-based Facility Explorer (FX)** as a lower/mid-tier line — dual-track strategy, distinct from core Metasys.
- **Market position**: Global tier-1, direct branches dominate delivery (less independent-dealer-driven than ALC), strong enterprise/security bundling (access control, video via C•CURE).
- **Takeaway for ALC dealers**: JCI's dual-track (proprietary core + Niagara FX) means the sales conversation must first establish which Metasys product is actually being quoted — the competitive posture is completely different for each.

## Honeywell (WEBs-N4/Niagara + separate EBI platform)
- **WEBs-N4** is Honeywell's Niagara 4-based mid-market platform: Java/HTML5 UI, BajaScript 2.0, semantic tagging, built-in auth/encryption defense-in-depth, RBAC, FIPS 140-2 conformance. The Spyder Model 5 controller line is marketed as "the first unitary controller freely programmable with Niagara 4," supporting online changes over MS/TP ([Honeywell WEBs-N4 overview](https://honeywelldynamic.hashtechorange.com/bms/building-control-system/webs-n4)).
- WEBs N4.8 Supervisor: FIPS 140-2 Level 1, FedRAMP RMF-accreditation-eligible, manages WEB-8000/CIPer/Niagara Edge devices plus Niagara Analytics ([WEBS N4.8 datasheet](https://hbtmkto.honeywell.com/rs/127-UCB-316/images/WEBS%20N4.8%20Supervisor%20Datasheet.pdf)).
- **EBI (Enterprise Buildings Integrator)** is a *separate*, older, higher-end enterprise platform — NOT Niagara-based. Server-client architecture, open protocols (OPC, Modbus, BACnet, LonWorks), a legacy HMIWeb/ActiveX browser client, and ODBC integration to SAP/PeopleSoft. Positioned for large campus/enterprise integration spanning BAS + fire + security + access ([Honeywell EBI overview](https://buildings.honeywell.com/us/en/solutions/integrated-operations/ebi); [EBI R500.1 overview](https://www.scribd.com/document/445237203/EBI-R500-1-EBIOverview-pdf)).
- **Market position**: Split personality — WEBs-N4 competes as a Niagara OEM in the mid-market; EBI competes at the very top of the enterprise-integration market, largely disconnected technically from WEBs-N4.
- **Takeaway for ALC dealers**: Never assume "Honeywell system" means one technology stack — always confirm WEBs-N4 vs. EBI before quoting integration scope; the legacy ActiveX client on EBI is a real operational pain point worth noting when positioning WebCTRL's modern browser client.

## Schneider Electric EcoStruxure Building Operation (EBO)
- SmartX AS-P/AS-B controllers, programmed via EBO software (v3.0+), offer a **dual programming model** — both Script *and* Function Block programming in the same environment, marketed by Schneider as "unique to the industry" ([SmartX AS-P spec sheet](https://iportal.se.com/Contents/docs/SmartX%20AS-P%20Server%20-%20SmartX%20Controller%20Specification%20Sheet.pdf)).
- Native BACnet, LonWorks, and Modbus support; native SOAP/REST Web Services.
- eCommission mobile app supports field commissioning of SmartX IP controllers via BACnet discovery over UDP 47808 ([eCommission manual](https://manuals.plus/m/70019fccd1facc383b684068f40258a522e25e152b9754be8ea60f95b1eef273.pdf)).
- Full programming still requires the EBO software suite — architecturally analogous to ALC's WebCTRL/EIKON coupling, i.e., not a "headless" open controller ([SmartX getting-started guide](https://industrialmonitordirect.com/blogs/knowledgebase/schneider-electric-smartx-as-p-controller-getting-started)).
- **Market position**: Strong in cloud-centric/analytics-forward deployments; a [LinkedIn HVAC controller comparison](https://www.linkedin.com/pulse/top-hvac-controller-companies-how-compare-them-ldbaf) explicitly groups Schneider with ALC as "cloud-centric management" leaders. A direct technician discussion comparing Schneider vs. ALC exists on Reddit: [Schneider Electric vs. Automated Logic](https://www.reddit.com/r/BuildingAutomation/comments/1ko6amt/schneider_electric_vs_automated_logic/).
- **Takeaway for ALC dealers**: EBO's dual Script/Function Block model appeals to programmers who want scripting flexibility — acknowledge this rather than arguing EIKON is strictly superior; the honest differentiator is EIKON's live in-place logic view, not raw programming-language flexibility.

## Delta Controls
- Front end has migrated toward **enteliWEB** (v4.2+), a newer web-based interface supplementing/replacing the legacy ORCAview/ORCAweb clients — adds BACnet device filtering and GCL+ sub-property support, can be installed alongside ORCA, and can BBMD-share a remote BACnet network as a foreign device (but cannot share the same Ethernet/BACnet-IP connection as ORCA simultaneously) ([enteliWEB 4.2 release notes](https://manuals.plus/m/0c3d9d43f38d109610a96d09c4550c93da77b45d22e71f0c5550d13eec44efb4)).
- Controller programming language is **GCL+**, Delta's proprietary graphical/scripting hybrid.
- **Market position**: Strong native-BACnet reputation, well-regarded among independent integrators; smaller footprint than the "big 4" (Siemens/JCI/Honeywell/Schneider) but respected for openness. [ControlTrends' 2023 review series](https://controltrends.org/hvac-smart-building-controls/building-automation-and-integration/08/building-automation-controls-systems-reviews-automated-logic/) began with Delta Controls before covering ALC, implicitly positioning them as comparable-tier "solid choice" vendors.
- **Takeaway for ALC dealers**: Delta's independent-dealer-heavy model mirrors ALC's — this is a genuine peer competitor in the "respected mid-tier, strong technical reputation" category rather than a top-tier global brand fight.

## Distech Controls (Eclypse series)
- **ECLYPSE** connected system controllers (ECY-APEX-BI, ECY-S1000-BI, ECY-VAV, etc.) are BTL-listed and BACnet-native ([BACnet Testing Labs Distech listing](https://www.bacnetinternational.net/btl/listings/distech%20controls,%20inc./BTL_Listing_31198_DistechControls_ECLYPSE.pdf)).
- Programming: **EC-gfxProgram**, a single common graphical drag-and-drop tool used across the entire ECL/ECB/ECLYPSE controller line, plus **ENVYSION** for web-based graphics/dashboards ([ECLYPSE Designer spec](https://blobanarus.blob.core.windows.net/rdbitzercoinc-distechcontrolsproducts/SDIDIDESIGNER01250_Spec.pdf)).
- Distech is **owned by Acuity Brands** but built its ecosystem heavily around **Niagara interoperability** — ECLYPSE controllers can be discovered and launched directly from Niagara Workbench via a "launch wizard," with live logic viewing and trend access, all stored back into the Niagara station database on backup ([Distech GFX/Niagara integration demo](https://www.youtube.com/watch?v=qK4gJz-cDHU)).
- Productivity tooling: Xpress gfx (auto-generates I/O from points lists), gfxApplications (pre-built AHU/FCU/VAV/RTU templates + 300 code snippets), myDC Control/AirBalancing (mobile commissioning/override app) — all aimed at reducing engineering time (per [Distech ECLYPSE demo deck](https://slideplayer.com/slide/10415006/)).
- **Market position**: A leading Niagara-native controller OEM; often specified where owners want vendor-neutral supervisory control with modern controller hardware — a direct architectural contrast to ALC's single-vendor stack.
- **Takeaway for ALC dealers**: Distech's live logic viewing inside Niagara Workbench is a real answer to "ALC's Logic tab is unique" — don't claim EIKON is the only platform with live in-place troubleshooting. The actual differentiator is single-vendor engineering tightness vs. Niagara's multi-vendor flexibility, not troubleshooting UX alone.

## Tridium Niagara Framework (ecosystem, not a single vendor)
- Niagara is licensed middleware, not a single company's end product — Tridium licenses the Niagara 4 framework and JACE hardware to OEMs (Honeywell WEBs-N4, JCI Facility Explorer, Distech ECLYPSE, Siemens TNM8000, Delta, KMC, Alerton, Lynxspring JENEsys Edge, etc.), each of whom can re-skin the UI ([r/BuildingAutomation "Differences in BMS" thread](https://www.reddit.com/r/BuildingAutomation/comments/1dvlmwj/differences_in_bms/)).
- **Licensing**: Niagara 4 uses a perpetual license with required SMAs (Software Maintenance Agreements) for version updates; Supervisor licensing is tiered by JACE connection count (1, 2, 3, 10, 100, unlimited) ([Niagara 101 presentation](https://www.tridium.com/content/dam/tridium/en/documents/events/2024-10-31-Niagara-101-ppt.pdf)).
- **Openness enforcement**: Niagara specs commonly *mandate* that "all controllers must be able to be programmed within the Niagara Workbench" and require open NiagaraNetwork Interface Component (NIC) statements — explicitly banning vendor lock-in via proprietary config tools. This is a structural, spec-driven vendor-neutrality that contrasts with ALC's single-vendor EIKON/WebCTRL stack ([Honeywell WEBs-N4 sample spec](https://www.smartbuildingdesign.com/wp-content/uploads/2025/02/Honeywell-WEBs-N4-Building-Management-System.pdf)).
- **Caveat on true openness**: a JACE only recognizes a protocol/device if the correct Tridium driver module is installed and licensed — so while the Workbench UI is standardized, protocol coverage itself is still gated by licensed modules (BAJA architecture) ([Smart Buildings Academy explainer](https://guides.smartbuildingsacademy.com/blog/tridium-part-2)). A vendor *can* restrict Workbench access to their own integrators if they configure NICs that way — openness is a specification choice, not an inherent guarantee.
- JACE 8000 hardware: runs Niagara 4.1+, requires SMA at purchase (18-month initial term, 3/5-year renewal options) ([JACE 8000 datasheet](https://www.tridium.com/content/dam/tridium/en/documents/document-lists/jace/2024-0028-JACE8000-DataSheet-Tridium.pdf)).
- **Market position**: The dominant "universal front end" for multi-vendor integration projects — frequently the default choice when an owner wants long-term vendor independence, at the cost of losing the single-vendor "seamless" engineering ALC/Siemens/JCI native stacks offer.
- **Takeaway for ALC dealers**: When a spec explicitly mandates Niagara Workbench programmability, that's a structural requirement ALC cannot meet as sole BAS — don't try to argue around it. The realistic play is ALC field controllers behind/alongside a JACE, never ALC as "the Niagara-compliant answer."

## KMC Controls & Alerton
- Both are longstanding native-BACnet controller manufacturers, frequently cited alongside Niagara-compatible product lines. A general industry roundup lists KMC as differentiated by "scalable solutions with strong cybersecurity features" ([LinkedIn HVAC controller comparison article](https://www.linkedin.com/pulse/top-hvac-controller-companies-how-compare-them-ldbaf)).
- Both integrate into the Niagara ecosystem as OEM/driver partners in various deployments (per the Niagara ecosystem list above), though neither has as deep a Niagara-native product identity as Distech or Honeywell WEBs-N4.
- Alerton's legacy BACtalk/Ascent platforms are native-BACnet from inception (Alerton was an early BACnet pioneer alongside ALC); market presence today is smaller/more regional compared to the top-tier vendors above.
- **Takeaway for ALC dealers**: These are typically encountered as existing installed base on takeover/integration jobs rather than head-to-head competitive bids — treat as native-BACnet gateway integrations (see `integration-pitfalls.md`), not competitive displacement targets.
